Problem Solving


Would you like to hear about that one time I got blocked on a simple problem?

What was the problem?

I was working on a JavaScript kata (A repetitive exercise a la "Wax on, wax off" in Karate Kid), and it was asking me to use a .map method which I'd not been exposed to yet.

What problem solving techniques did I use?

Firstly I looked the method up on Google. I hated the first example because it was super abstract and hard to wrap my head around, so I tried another couple of places, including some random people's blogs as I figured they might have some more.. ground level.. examples. That kind of worked, and I felt the germ of understanding start to grow.

I then copy/pasted an example off Google into my own code, and set about changing it so that it would work in my function. Contextualising the information like that helped with my understanding. It took some trial and error, but I got there in the end.

How did I feel throughout the process?

Sometimes frustrated, to be honest, but a lot of the time I was just up for the challenge. At my worst moments, I was fearing that I had accidentally skipped over some rudimental piece of information which would have made all of this a lot easier to understand.

What did I learn?

In this instance I learned what a Fat Arrow does, which has opened up a bit of a door in terms of general understanding I think.


Would you like to hear another story? This one is about that time I solved a problem elegantly.

What was the problem?

In the very next kata, I was asked to use a .find method, which I had also never used before.

What problem solving techniques did I use?

Again, I googled the method and this time I noticed that in the example, the syntax of the method was very similar to the one for .map, so I applied what I had already learned, and was able to get the .find exercise to work quickly. Recognising similarities between problems and letting those guide you to the solution faster is a useful problem solving tool.

How did I feel throughout the process?

Vaguely confident, but still keeping on my toes. You never know when an unnoticed typo will strike, or if it'll turn out that what you think you know turns out to be wrong. (Fun fact: I originally had a typo in that bit about unnoticed typos.. *sad trumpet*)

What did I learn?

Once you start noticing similarities between different methods, every time you encounter a new concept it's not an enormous monster you have to figure out from scratch. Things start to seem a bit easier.
